# Restricted Netlist Generators

This page explains why Circuit Netlist uses restricted generation models (`repeat`, etc.) rather than arbitrary code execution.

<Aside type="note" title="Boundary">

This page only explains why the Web UI is generated in a restricted manner.
For syntax and editor/save contracts, see Reference.

</Aside>

## Design reason

- **Deterministic**: The same Source Form must be stably expanded into the same Expanded Form.
- **Debuggable**: Syntax errors and expansion errors can be located in specific blocks.
- **Minimal Surface Area**: Avoid turning the schema editor into a script execution environment.

## M1 support range

- `repeat`
- `count` / `start`
- `index` / `symbol` / `series`
- Restricted template interpolation (`${index}`, `${symbol}` with fixed offset)

## non-target

- Any `for/if`
- Nested `repeat`
- Any function call
