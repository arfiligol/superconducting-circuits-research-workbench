# Data Formats

This chapter is the SoT of the persisted research model, with the main axes:

- `DatasetRecord` is the top-level container of collaboration and session context
- `DesignScope` is a dataset-local lifecycle-managed analytical scope
- `TraceRecord` is the standard trace unit that can be analyzed
- `TraceBatchRecord` is the setup / provenance / lineage boundary
- `TraceStore` (Zarr) holds large ND numeric payload
- `AnalysisRunRecord` + `ResultArtifactRecord` define the artifact-first result view

<Aside type="caution" title="Dataset-first baseline">

- `active_dataset` binds `DatasetRecord`, not design.
- For `Raw Data Browser` and `Characterization`, dataset-local `design_id` is selected.
- Data Ingestion and Simulation publication must explicitly select the existing `DesignScope` or create-new target.
- Characterization inputs are always compatible S/Y/Z matrix traces.
- `layout_simulation` / `circuit_simulation` / `measurement` are just from different sources.
- metadata DB and numeric TraceStore must be separated.
- canonical trace can be 1D / 2D / ND; sweep point is not the only canonical record unit.

</Aside>
