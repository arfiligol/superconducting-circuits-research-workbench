# Session & Workspace

This page defines the runtime-mode-aware session / workspace authority surface shared by frontend shell, workspace pages and shared app model.

<Aside type="note" title="Surface Boundary">

This page owns runtime mode, session, workspace membership, active workspace, active dataset, workspace invitation / membership management, user summary, workspace role, and capability exposure.
task lifecycle, definition preview, result artifact and audit query are not the responsibility of this page.

</Aside>

<Aside type="caution" title="Single Context Authority">

`Active Workspace`, `Active Dataset`, `Workspace Role`, `User Menu Summary` displayed by app shell
Must come from the same backend session authority as collaboration controls.

</Aside>

<Aside type="caution" title="Local And Online Must Not Share One Session Cache">

Both local mode and online mode use the same session surface, but they must not share the same active context.
The mode switch must re-establish the session envelope instead of inheriting the workspace/dataset/user state of the old mode.

</Aside>

<Aside type="note" title="Authorization baseline">

All capability flags, membership mutation permission and collaboration control on this page are subject to the backend authorization engine.
The official baseline adopts `Casbin`. The backend service first sorts out the resource envelope and then returns materialized `capabilities` and `allowed_actions`.

</Aside>

## Coverage

| Surface | Meaning |
| --- | --- |
| Runtime mode | app is currently running as `local` or `online` |
| Auth state | Whether the user is currently `local_bypass`, `authenticated`, `anonymous` or `degraded` session |
| Connection target | A summary of the currently active online target, or `null` |
| User summary |Header user icon / menu required display data|
| Workspace identity |active workspace, membership role, default task scope|
| Workspace membership list | user switchable workspaces and roles |
| Active dataset | shell-level global dataset context |
| Capability flags | frontend and shared task-management directly usable permission summary |
| Collaboration mutations |invite, revoke, leave, remove member, transfer owner|

## Session Envelope

The backend session surface must provide at least:

| Field | Meaning |
| --- | --- |
| `auth.state` |`local_bypass`, `authenticated`, `anonymous`, `degraded` and similar values session state|
| `runtime_mode` | `local` or `online` |
| `auth.mode` | auth transport or mode summary |
| `session_id` | Current session identity; public online session can be `null` |
| `connection.target` | Nested target summary object containing `origin`, `label`, `is_active`, `validation_status`, `last_checked_at`; local mode uses `null` |
| `user.id` | Current user identity |
| `user.display_name` | Header user icon / menu display name |
| `user.platform_role` | `admin` / `user` |
| `workspace.memberships[]` | Workspace list and role summary that users can enter; local mode can be reduced to a single `Local Space` membership |
| `workspace.id` | active workspace identity |
| `workspace.slug` | workspace route / slug summary |
| `workspace.name` | workspace display name |
| `workspace.role` | `owner` / `member` / `viewer` |
| `workspace.default_task_scope` | task default visibility scope |
| `workspace.allowed_actions` | workspace-scoped mutation availability summary |
| `active_dataset.id` / `name` | global dataset context |
| `active_dataset.visibility_scope` | One of `local`, `private`, `workspace` |
| `capabilities` | capability flags summary |

## Runtime Mode Rules

| Rule | Meaning |
| --- | --- |
| Same envelope family | local and online should return compatible session envelope shapes |
| Local mode returns implicit session | backend directly provided local user / `Local Space` / capability summary |
| Online mode may require auth | If you have not logged in, the session surface will return `auth_required` or anonymous/degraded state |
| Mode switch is session mutation | Not just frontend config toggle |
| Mode switch clears stale shell context | The dataset, task list, and attached task refs of the old mode must not remain |
| Mode switch drops remote auth continuity | Switch from online to local and back to online without retaining the old remote authenticated session |

## Authorization Resolution Rules

| Rule | Meaning |
| --- | --- |
| Session never trusts JWT claims for permissions |JWT only provides authentication continuity|
| Workspace role is backend-resolved | Active workspace role is decided by membership lookup + authorization engine |
| Capability flags are materialized decisions | frontend does not need to rebuild the role matrix by itself |
| Mutation availability is echoed | invite / revoke / leave / remove / transfer and other controls must return `allowed_actions` or equivalent flags |
| Casbin policy is implementation baseline | policy engine can be replaced, but the contract exposed by frontend remains unchanged |

## Active Workspace Switching

| Rule | Meaning |
| --- | --- |
| Session exposes memberships | Header should be able to list user-switchable workspaces |
| Switch mutates session |Active workspace switch is a session mutation, not a pure frontend state|
| Dataset rebinding is required | After the workspace is switched, the active dataset must be re-verified or re-selected |
| Task rebinding is required | After the workspace is switched, task visibility and allowed actions must be updated simultaneously |

## Mutation Surfaces

| Mutation | Responsibility |
| --- | --- |
| `switch_runtime_mode(runtime_mode, server_origin | null, label | null)` | Switch `local` / `online` to establish corresponding session envelope or auth-required outcome |
| `list_server_targets()` | List active targets and remembered recent online targets |
| `validate_server_target(server_origin)` | Validates target reachability / health / version compatibility |
| `remember_server_target(server_origin, label | null)` | Save or update app-level online target config |
| `switch_active_workspace(workspace_id)` | Change active workspace, return new session envelope and rebind outcome |
| `activate_dataset(dataset_id | null)` | Set the active dataset and verify that the dataset is visible to the current active workspace |
| `create_workspace_invitation(workspace_id, email, role)` | Create pending invite and trigger outbound delivery |
| `revoke_workspace_invitation(invite_id)` | Withdraw pending / delivered invite before accept |
| `accept_workspace_invitation(invite_token)` | Verify invite, establish membership, and return post-accept suggestion |
| `leave_workspace(workspace_id)` | The user actively exits the membership and reconnects the session context |
| `remove_workspace_member(workspace_id, member_user_id)` | Remove other memberships |
| `transfer_workspace_owner(workspace_id, new_owner_user_id)` | Transfer owner permissions and return updated membership summary |

## Live HTTP Routes

| Route | Status | Responsibility |
| --- | --- | --- |
| `GET /session` | live |Envelope runtime-mode-aware session envelope|
| `PATCH /session/runtime-mode` | live | Switch `local` / `online` and return new session envelope + switch result fields |
| `GET /session/server-targets` | live | List remembered online targets and active target origin |
| `POST /session/server-targets` | live | Add or update app-level online target config |
| `POST /session/server-targets/validate` | live | Validates target reachability / health / compatibility |
| `PATCH /session/active-workspace` | live |Switch active workspace|
| `PATCH /session/active-dataset` | live | switch active dataset |
| `POST /session/login` | live |Create online authenticated session|
| `POST /session/logout` | live |Clear online authenticated session|
| `POST /session/refresh` | live | Rebuild online authenticated session via refresh transport |

## Workspace Switch Response

| Field | Meaning |
| --- | --- |
| `workspace.id` / `name` / `role` |new active workspace|
| `active_dataset_resolution` | `preserved`, `rebound`, `cleared` |
| `active_dataset` | The rejoined dataset summary, or `null` |
| `detached_task_ids[]` | Task ids detached due to visibility failure |
| `capabilities` | capability summary under new workspace |
| `memberships[]` | For Header to re-render switcher |

## Runtime Mode Switch Response

<Aside type="note" title="Extends the session envelope">

The successful response of `PATCH /session/runtime-mode` is not a separate small payload.
It returns the complete session envelope, plus additional runtime-mode switch result fields.

</Aside>

| Field | Meaning |
| --- | --- |
| `runtime_mode` | New active mode |
| `connection.target` | nested target summary object or `null`; plain string target is no longer used |
| `outcome` | backend switch outcome: The current live values â€‹â€‹are `entered_local`, `entered_online_auth_required` |
| `auth_transition` |auth continuity outcome: The current live values â€‹â€‹are `entered_local_bypass`, `online_auth_required`, `online_session_dropped`|
| `session_reset` | Whether the old mode shell context has been cleared |
| `workspace` | Use standard session workspace object; online public session also retains object shape, but identity fields are `null` |
| `active_dataset` | Use the standard session active-dataset object, or `null` if the dataset has not been parsed yet |
| `capabilities` | capability summary by new mode materialize |
| `detached_task_ids[]` | Task ids that are detached when mode switch |

## Server Target Summary Shape

| Field | Meaning |
| --- | --- |
| `origin` |canonical remote origin, such as `https://lab.example.com`|
| `label` | target display label |
| `is_active` | Is it the current active online target |
| `validation_status` |`validated`, `target_validation_failed`, `target_unreachable`, `target_incompatible`, `online_target_rejected`|
| `last_checked_at` | The latest validation timestamp; if it has not yet been verified, it can be `null` |

## Online Server Target Rules

| Rule | Meaning |
| --- | --- |
| App-level owner | server target config does not belong to workspace, dataset, or account preference |
| One active target | There is only one active online target at the same time |
| Recent history allowed | remembered recent targets can be retained for reuse by mode switcher |
| Validation before switch | Target validation must be completed before switching to online mode |
| Failure keeps current mode | If target verification fails, keep the original active mode and do not establish a half-completed session |
| Connection shape | `connection.target` in session / runtime-mode response Always use nested summary object or `null` |

## Active Dataset Activation Rules

| Rule | Meaning |
| --- | --- |
| Must belong to active workspace | If the dataset does not belong to or is not visible in the active workspace, the mutation must be rejected |
| Explicit clear allowed | Allow active dataset to be set to `null` |
| Session-wide propagation | After mutation is successful, all page consumers see the same active dataset |
| Rejection must explain why | At least distinguish `not_found`, `not_visible_in_workspace`, `archived` and other reasons |

## Local Session Baseline

| Field | Required behavior |
| --- | --- |
| `runtime_mode` | Must be `local` |
| `auth.state` | Use `local_bypass` or equivalent local state |
| `user` | Return implicit local operator summary |
| `workspace` | Return `Local Space` |
| `memberships[]` | Can be omitted, or return a flyer `Local Space` membership |
| `connection.target` | Returns `null`, not `"local"` string |
| `active_dataset.visibility_scope` | If the local dataset is bound, it must be `local` |
| `capabilities` | Return local mode capability summary, not guessed by frontend |

<Aside type="note" title="Local mode still uses the same backend surface">

Local mode does not mean skipping the backend session surface.
The frontend should still obtain user summary, workspace, dataset and capability flags through the same authority.

</Aside>

## Invitation Surfaces

| Field | Meaning |
| --- | --- |
| `invite_id` | invite identity |
| `workspace_id` | target workspace |
| `email` | invited email |
| `role` | proposed workspace role |
| `status` | `pending`, `delivered`, `accepted`, `revoked`, `expired`, `delivery_failed` |
| `expires_at` | invite expiry |
| `delivery.channel` | `smtp` or `manual_link` |

### Invitation acceptance response

| Field | Meaning |
| --- | --- |
| `invite.status` | `accepted`, `revoked`, `expired`, `rejected_account_mismatch` |
| `workspace` | Invited workspace summary |
| `membership.role` | New membership role |
| `switch_available` | If there are other active workspaces, prompt frontend display switching CTA |
| `recommended_next_action` | `switch_workspace`, `stay_current`, `reopen_invite`, etc. |

## Membership Management Rules

| Rule | Meaning |
| --- | --- |
| Invite creation is explicit | There is no silent auto-join |
| Owner leave requires another owner | Transfer ownership |
| Removing active member rebinds session | If the target user is using this workspace, the session must be cleared active workspace / dataset |
| Transfer owner updates capabilities immediately | After the owner transfer is successful, the capability summary of both parties must be changed immediately |

## Error Code Contract

| Code | Category | When it applies |
| --- | --- | --- |
| `auth_required` | `auth_required` | Invalid session or not logged in |
| `request_validation_failed` | `validation_error` | `runtime_mode`, `server_origin`, `label`, etc. request payload is illegal |
| `target_validation_failed` | `conflict` | The target response has validation failure and cannot enter online mode |
| `target_unreachable` | `conflict` | online server target cannot connect |
| `target_incompatible` | `conflict` | target reachability is normal, but version or contract is incompatible |
| `online_target_rejected` | `conflict` | target validation was rejected or could not be classified into a more detailed failure code |
| `auth_session_expired` | `auth_required` | The previous online authenticated session has expired and needs to be logged in again |
| `workspace_membership_required` | `permission_denied` | Target workspace does not belong to current memberships |
| `dataset_not_visible_in_workspace` | `permission_denied` | activate dataset points to invisible dataset |
| `dataset_archived` | `conflict` | Active dataset is archived |
| `workspace_invite_expired` | `conflict` | invite token has expired |
| `workspace_invite_revoked` | `conflict` | invite revoked |
| `workspace_invite_account_mismatch` | `permission_denied` | invite email does not match the current account |
| `workspace_invite_create_denied` | `permission_denied` | session without invite permission |
| `workspace_invite_not_found` | `not_found` | invite identity does not exist |
| `workspace_member_remove_denied` | `permission_denied` | session does not have permission to remove members |
| `workspace_owner_transfer_denied` | `permission_denied` | session without transfer owner permission |
| `workspace_owner_transfer_invalid` | `validation_error` | The target user does not meet the owner transfer conditions |
| `context_rebind_required` | `conflict` | You need to reselect dataset/task after session switching |

## Delivery Rules

| Rule | Meaning |
| --- | --- |
| Dataset switch is global | active dataset Once switched, all page consumers should see the same version |
| Mode switch is stronger than workspace switch | After switching mode, workspace / dataset / task list / user summary must be rebuilt in the new mode |
| Role is workspace-scoped | The action availability seen by shells and pages must be determined by the workspace role |
| Capability beats guessing | frontend page Do not infer whether it is possible submit / manage / delete |
| Session survives refresh | User summary, workspace role and active dataset must be rebuilt after refresh |
