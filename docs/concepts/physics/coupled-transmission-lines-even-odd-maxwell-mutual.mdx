# Coupled Transmission Lines：Even/Odd、Maxwell、Mutual

如果你現在想做的是 distributed coupled-line section，幾乎一定會撞到這三種表示法：

- even / odd mode 參數
- Maxwell capacitance matrix
- mutual capacitance matrix

這三個名字看起來像三套不同理論，但其實它們在討論的是**同一段對稱雙導體 coupled line**，只是各自站在不同的描述角度。

這篇的核心問題是：

**為什麼 `JosephsonCircuits` 會同時提供 `maxwell_to_even_odd`、`mutual_to_even_odd`、`even_odd_to_mutual`？如果我要寫 `add_coupled_line_section!`，到底該用哪一個？**

## 先講結論

如果你的目標是**真的把 coupled line 寫成 netlist 裡的一段 distributed ladder**，最實用的流程通常是：

1. 先用 even/odd 參數描述你要的 coupled section
2. 用 `even_odd_to_mutual(...)` 轉成 `L` 與 `C_mutual`
3. 再把它離散成很多個小 section
4. 每個 section 都用：
   - 兩顆 series `L`
   - 一個 mutual `K`
   - 兩顆對地 `C`
   - 一顆跨線 `C`

也就是說：

- **even/odd form** 最適合做規格與物理解讀
- **Maxwell form** 最適合做矩陣分析
- **mutual form** 最適合映射到「實體 netlist 元件」

如果你最後是要寫 `add_coupled_line_section!`，最關鍵的通常不是 `maxwell_to_even_odd`，而是：

- `even_odd_to_mutual`
- `mutual_to_even_odd`

前者把 modal specification 變成可下 netlist 的 physical section；後者則讓你檢查「我寫出的 physical ladder，是不是對應我原本想要的 even/odd 模態」。

## 你可以從這裡開始

- 你知道 transmission line 的 `Z_0`、相速度、相位速度或折射率
- 你知道 coupled line 會有 even mode 與 odd mode
- 你能接受 2×2 矩陣表示每單位長度的 `L` 與 `C`

若要補背景，請先讀：

- [Physics Overview](index.mdx)
- [Schur Complement & Kron Reduction](schur-complement-kron-reduction.mdx)

如果你只想先做任務：

## 三種表示法在講同一件事

### 1. Even / Odd mode form：最適合講物理

對稱雙線 coupled line 的兩個本徵模態是：

- **even mode**：兩條線同電位、同方向電流
- **odd mode**：兩條線反相電位、反方向電流

對一段對稱 coupled line，你常常拿到的是：

- `Z_even`
- `Z_odd`
- `n_even`
- `n_odd`

這四個量的好處是物理直覺很強：

- `Z_even` / `Z_odd` 告訴你兩個模態的特性阻抗
- `n_even` / `n_odd` 告訴你兩個模態的相速度差異

這也是 Qucs 對 coupled transmission line 元件的主要描述方式之一：把 coupled line 看成 even mode 與 odd mode 各自一條獨立的傳輸線，再組回 4-port 結果。參見 Qucs 的 coupled transmission line technical note。[Qucs](https://qucs.sourceforge.net/tech/node63.html)

### 2. Maxwell capacitance matrix：最適合做節點電荷-電壓分析

`JosephsonCircuits.maxwell_to_even_odd(L, Cmaxwell)` 用的是 **Maxwell capacitance matrix**。

它的定義是：

```text
Q = C_maxwell V
```

也就是說：

- `C_maxwell[i, j] = dQ_i / dV_j`
- 對角項代表該節點電荷對自己電壓的靈敏度
- 非對角項通常是負值，因為鄰近節點電壓上升會拉走本節點的電荷

對兩條對稱 coupled line，如果用 Maxwell form，最典型會長成：

```text
C_maxwell =
[ C11   C12 ]
[ C12   C11 ]
```

而且通常 `C12 < 0`。

在 `JosephsonCircuits` 的 reference 裡，對稱情況下有：

```text
Z_even = sqrt((L11 + L12) / (C11 + C12))
Z_odd  = sqrt((L11 - L12) / (C11 - C12))
```

見官方 reference 與原始實作。[JosephsonCircuits Reference](https://josephsoncircuits.org/stable/reference/)

### 3. Mutual capacitance matrix：最適合畫成實體 netlist

`JosephsonCircuits.even_odd_to_mutual(...)` 回傳的是 **mutual capacitance matrix**。

這個表示法的重點不是 `Q = CV`，而是：

- 對角項 = 每條線對地的實體電容
- 非對角項 = 線與線之間的實體耦合電容

也就是說，它更像「真正要在 netlist 裡擺的電容元件表」。

對稱雙線時，它長得像：

```text
C_mutual =
[ Cg   Cm ]
[ Cm   Cg ]
```

這裡：

- `Cg` 是每條線對地的 physical capacitor
- `Cm` 是兩條線之間的 physical coupling capacitor

這就是為什麼 **mutual form 最適合直接變成 ladder section**。

## 為什麼 Maxwell form 與 mutual form 長得不一樣

這是最容易混淆的地方。

在 `JosephsonCircuits` 裡：

```text
C_mutual = maxwell_to_mutual(C_maxwell)
```

對稱雙線最常見的對應是：

```text
C_mutual =
[ Cg   Cm ]
[ Cm   Cg ]
```

對應到：

```text
C_maxwell =
[ Cg + Cm   -Cm     ]
[ -Cm       Cg + Cm ]
```

所以：

- mutual form 的 `Cm` 是你真的會放在兩線之間的 capacitor
- Maxwell form 的 off-diagonal 是負的，因為它是節點電荷-電壓關係，不是 physical branch list

這也是為什麼 `mutual_to_even_odd(...)` 的結果會變成：

```text
Z_even = sqrt((Ls + Lm) / Cg)
Z_odd  = sqrt((Ls - Lm) / (Cg + 2Cm))
```

乍看很奇怪，但其實完全合理：

- even mode 時，兩條線同電位，所以**跨線電容 `Cm` 沒有電壓差，不參與儲能**
- odd mode 時，兩條線反相，所以**跨線電容兩端電壓差最大，儲能最強**

因此：

- even mode 看到的有效電容只剩 `Cg`
- odd mode 看到的有效電容會增加到 `Cg + 2Cm`

這正是 `JosephsonCircuits.mutual_to_even_odd(...)` 的物理意義。[JosephsonCircuits Reference](https://josephsoncircuits.org/stable/reference/)

## 用基底轉換看這件事，會最清楚

令 even/odd basis 的變換矩陣為：

```text
T = 1/sqrt(2) * [ 1   1 ]
                  [ 1  -1 ]
```

對稱情況下，

```text
L =
[ Ls   Lm ]
[ Lm   Ls ]
```

會被對角化成：

```text
T^T L T = diag(Ls + Lm, Ls - Lm)
```

而 Maxwell capacitance matrix

```text
C_maxwell =
[ Cg + Cm   -Cm     ]
[ -Cm       Cg + Cm ]
```

也會被對角化成：

```text
T^T C_maxwell T = diag(Cg, Cg + 2Cm)
```

所以 even / odd 模態本質上就是把對稱 coupled pair 對角化。

這也是 `maxwell_to_even_odd` 看起來那麼簡潔的真正原因：
它不是神奇公式，而是**對稱雙線的 2×2 模態分解**。

## 三個函式各自該怎麼用

### `maxwell_to_even_odd(L, Cmaxwell)`

用在你已經有：

- inductance matrix `L`
- Maxwell capacitance matrix `Cmaxwell`

而你想知道：

- `Z_even`
- `Z_odd`
- `n_even`
- `n_odd`

這很適合：

- 從 field solver / nodal matrix 結果抽 modal quantities
- 做基底理解與 sanity check

### `mutual_to_even_odd(L, Cmutual)`

用在你已經有：

- physical inductor coupling
- physical capacitors to ground and between lines

而你想知道：

- 這些實體元件對應到什麼 even / odd 模態

這很適合：

- 檢查你手刻的 ladder section 是不是合理
- 從實體 section 值回推 modal behavior

### `even_odd_to_mutual(Zeven, Zodd, neven, nodd)`

這是如果你要寫 `add_coupled_line_section!` 最關鍵的一個。

它適合的問題是：

**我先在模態空間指定 coupled line 規格，怎麼把它變成 netlist 裡的 section 元件？**

也就是：

```text
(Z_even, Z_odd, n_even, n_odd)
    -> (L matrix, C_mutual matrix)
    -> ladder sections
```

## 例子 1：完全不耦合的極限

如果

```text
Lm = 0
Cm = 0
```

那麼：

```text
L = [Ls 0; 0 Ls]
C_mutual = [Cg 0; 0 Cg]
```

此時：

- `Z_even = Z_odd`
- `n_even = n_odd`

這就是兩條互不影響的獨立線。

這個例子很重要，因為它是所有 coupled-line builder 的第一個 sanity check：

- 如果把 coupling 關掉，結果必須退化回兩條普通 transmission line

## 例子 2：為什麼 odd mode 比 even mode 更慢

假設：

```text
Ls = 1.0
Lm = 0.1
Cg = 1.6
Cm = 0.4
```

那麼：

```text
L_even = Ls + Lm = 1.1
L_odd  = Ls - Lm = 0.9
C_even = Cg      = 1.6
C_odd  = Cg + 2Cm = 2.4
```

所以：

- even mode 的電感比較大，但電容較小
- odd mode 的電感比較小，但電容更大

在很多實際結構裡，odd mode 的電容增加更明顯，因此常會比 even mode 更慢。

這也就是為什麼 coupled-line calculator 經常會同時報：

- `Z_even`
- `Z_odd`
- `n_even`
- `n_odd`

而不是只給你單一 `Z0`。

## 例子 3：從 even/odd 規格回推出單一 ladder section

假設你有一段 coupled section，長度為 `ℓ`，分成 `N` 段。

你已知：

- `Z_even`
- `Z_odd`
- `n_even`
- `n_odd`

先做：

```julia
Lmat, Cmut = JosephsonCircuits.even_odd_to_mutual(Zeven, Zodd, neven, nodd)
```

得到每單位長度矩陣：

```text
Lmat =
[ L11  L12 ]
[ L12  L11 ]

Cmut =
[ Cg   Cm ]
[ Cm   Cg ]
```

令 `Δx = ℓ / N`，則每一個離散 section 可以寫成：

```text
Line A series inductor:  L11 * Δx
Line B series inductor:  L11 * Δx
Mutual coupling:         K = L12 / L11
Line A shunt capacitor:  Cg  * Δx
Line B shunt capacitor:  Cg  * Δx
Cross capacitor:         Cm  * Δx
```

如果是 lossless superconducting case，則：

```text
R' = 0
G' = 0
```

完全不需要再放任何 `R` 或 `G` branch。

## 例子 4：我們怎麼做 `add_coupled_line_section!`

下面是一個**概念上正確、而且和本 repo 目前 netlist 風格一致**的做法：

```julia
function add_coupled_line_section!(
    circuit;
    prefix,
    left_node_a,
    right_node_a,
    left_node_b,
    right_node_b,
    length_m,
    n_sections,
    Zeven,
    Zodd,
    neven,
    nodd,
)
    Lmat, Cmut = JosephsonCircuits.even_odd_to_mutual(Zeven, Zodd, neven, nodd)
    dx = length_m / n_sections

    Lself = Lmat[1, 1] * dx
    Lmut = Lmat[1, 2] * dx
    Cg = Cmut[1, 1] * dx
    Cm = Cmut[1, 2] * dx
    k = Lmut / Lself

    # 然後對每一段 section：
    # 1. line A 放一顆 series L
    # 2. line B 放一顆 series L
    # 3. 兩顆 L 之間放一個 K
    # 4. 兩條線各放一顆對地 Cg
    # 5. 兩條線之間放一顆 Cm
end
```

如果展成單一 section，它大概會對應到這樣的 netlist：

```text
("L_A_1", a0, a1, Lself)
("L_B_1", b0, b1, Lself)
("K_1", "L_A_1", "L_B_1", k)
("C_A_1", a1, 0, Cg)
("C_B_1", b1, 0, Cg)
("C_AB_1", a1, b1, Cm)
```

這就是為什麼我前面說：
**要寫 distributed coupled-line section，最實用的入口往往是 `even_odd_to_mutual`。**

## 為什麼這個 builder 是合理的

因為它同時滿足三件事：

1. **modal specification 清楚**
   - 你可以直接對 `Z_even`, `Z_odd`, `n_even`, `n_odd` 做設計與比對
2. **netlist mapping 清楚**
   - 你最後真的能用 `L / C / K` 元件落到 `JosephsonCircuits`
3. **可離散、可收斂**
   - `N` 越大，對 distributed section 的近似越好

這跟把 coupled line 硬塞成「某個神祕的 lumped coupler」不同。
你保留了：

- 長度
- 位置
- 局部耦合窗口
- 與其他結構的接點

所以它很適合用在你現在想做的事情：

- Purcell filter 的某一段與另一個結構做 distributed coupling
- readout line 與 resonator 做有限長度的 coupled window

## 工程映射：從模態規格到電路 builder

如果你是從設計角度出發，最自然的工作流通常是：

1. 先從 EM / calculator 拿到：
   - `Z_even`, `Z_odd`
   - `n_even`, `n_odd`
2. 用 `even_odd_to_mutual(...)` 得到 per-unit-length `L` 與 `C`
3. 選一個 coupled window 長度 `ℓ`
4. 選 section 數 `N`
5. 離散成 ladder
6. 再把這段 coupled ladder 插到更大的電路裡

如果你是從已知電路出發，想驗證某段耦合 window 的物理意義，則流程反過來：

1. 先寫出 section 的 physical `L / C / K`
2. 算出對應的 `Lmat`, `Cmut`
3. 再用 `mutual_to_even_odd(...)` 回推 modal quantities
4. 看它是不是你原本想要的 `Z_even/Z_odd/n_even/n_odd`

## 限制與近似

這篇的討論有幾個重要限制：

1. **只討論對稱的兩線 coupled section**
   - 也就是 `L11 = L22`、`C11 = C22`
2. **先假設 lossless**
   - `R' = 0`, `G' = 0`
3. **先用 TEM / quasi-TEM intuition**
   - 這對 CPW / microstrip 的低損近似很常見，但不是所有頻段都完美成立
4. **builder 是 ladder approximation，不是連續解析解本身**
   - `N` 太小時，分散式效果會被離散誤差污染

另外還要特別注意：

<Aside type="caution" title="文獻中的 C 矩陣慣例可能不同">

有些文獻的 `C_m` 是 telegrapher equation 中的耦合係數，有些文獻的 `C_m` 則是 physical capacitor。
在本專案裡，如果你要下 netlist，最安全的做法是明確區分：
- `C_maxwell`
- `C_mutual`

</Aside>

## 你可以怎麼向別人報告這件事

如果你要做口頭報告，最容易讓聽眾抓到主線的說法是：

1. **even/odd form**：是 coupled line 的模態語言，最適合談物理
2. **Maxwell form**：是電荷-電壓矩陣語言，最適合談節點分析
3. **mutual form**：是 physical capacitor / inductor 的語言，最適合下 netlist
4. 我們真正做 builder 時，是用 `even_odd_to_mutual` 把 modal spec 轉成可離散的 ladder section

如果你只記一件事，請記：

> `even_odd_to_mutual` 不是一個額外的花俏函式，它其實是在幫你把「模態規格」翻譯成「可實作的電路 section」。
