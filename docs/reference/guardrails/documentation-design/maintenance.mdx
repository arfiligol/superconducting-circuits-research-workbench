# Documentation Maintenance

This document defines the file maintenance process. `docs/` is the sole editorial source of high-level technical docs; Astro + Starlight are responsible for building it into `/docs/`; Sphinx and Documenter.jl are responsible for sibling API reference sites.

## Source Tree Map

| Location | Role |
| --- | --- |
| `docs/start/` | shortest onboarding path |
| `docs/workflows/` | task-first workflow docs |
| `docs/concepts/` | mental models, physics concepts, architecture concepts, design decisions |
| `docs/reference/` | stable contracts, API landing, guardrails, agent skills, contributors |
| `docs/contribute/` | contribution workflow entrypoints |
| `docs/api-reference/` | Sphinx and Documenter.jl API docs source |
| `site/src/content/docs/docs/` | generated Starlight staging tree |
| `site/dist/docs/` | built Starlight docs output |
| `build/api-reference/` | generated API docs output before public artifact copy |
| `site/dist/api/` | deployed API docs artifact path |

<Aside type="caution" title="maintenance rules">

- Daily editing will be done at `docs/`.
- `docs/start/`, `docs/workflows/`, `docs/concepts/`, `docs/reference/`, `docs/contribute/` is active Starlight source layout.
- Direct modification of `site/src/content/docs/docs/` is prohibited as it will be overwritten by `scripts/build/prepare_starlight_docs.py`.
- API reference source is placed in `docs/api-reference/` and must not be placed in Starlight staging.
- After adding, deleting or moving pages, you must run route validation and Starlight build.

</Aside>

## Frontmatter Update

| Change type | Minimum update |
| --- | --- |
| content changed | `last_updated`, `updated_by` |
| rule/page scope changed | `version` minor or major bump |
| typo / formatting only | patch bump |

When the content changes, at least update:

- `last_updated`: `YYYY-MM-DD`
- `updated_by`: `team` or `team/person`

## Astro + Starlight English Docs Build

The current official document structure name of this project is:

- `Astro + Starlight English Docs Build`

Its technical characteristics are:

- The main source tree is `docs/`
- Editable docs source is English-only.
- Astro + Starlight built into `site/` `/docs/`
- `scripts/build/prepare_starlight_docs.py` generates Starlight staging
- generated API reference deployed to sibling sites `/api/python/` and `/api/julia/`
- public artifact released by `site/dist/`

## Check process

<Tabs>

uv run docs-site serve --build

<TabItem label="Build inspection">


```bash
uv run python scripts/check_docs_nav_routes.py --check-source
uv run python scripts/check_docs_language.py
./scripts/build_docs_sites.sh
uv run python scripts/check_docs_nav_routes.py --check-built

# API reference
./scripts/build/build_api_docs.sh
```

</TabItem>

</Tabs>

<Aside type="tip" title="FAQ">

- If the page is still 404 after updating, first confirm whether `./scripts/build_docs_sites.sh` regenerates Starlight staging.
- If Starlight build fails, first look at the corresponding staging file in `site/src/content/docs/docs/`, and then go back to `docs/` to repair the source.
- If the API reference link fails, run the complete `./scripts/build/build_public_site.sh` and confirm that `/docs/`, `/api/python/`, and `/api/julia/` exist at the same time.

</Aside>

## Agent Rule
```markdown
## Documentation Maintenance
- **Docs Source**: edit `docs/` only.
- **Docs IA**: keep editable docs in the reader/task-first source layout: `docs/start/`, `docs/workflows/`, `docs/concepts/`, `docs/reference/`, and `docs/contribute/`.
- **Staging Tree**: `site/src/content/docs/docs/` is generated by `scripts/build/prepare_starlight_docs.py`; do not edit directly.
- **API Docs Source**: edit Sphinx and Documenter.jl API docs under `docs/api-reference/`; generated output goes through `build/api-reference/` and `site/dist/api/`.
- **Build Owner**: Astro + Starlight renders `/docs/`; Sphinx renders `/api/python/`; Documenter.jl renders `/api/julia/`.
- **Language**: editable docs source is English-only; generated/staging output remains excluded from direct editing.
- **Frontmatter**: update `last_updated` and `updated_by` on content changes.
- **Versioning**: patch/minor/major bumps for doc changes.
- **Architecture Term**: this repo uses `Astro + Starlight English Docs Build`.
- **Verify**: run `uv run python scripts/check_docs_nav_routes.py --check-source`, `uv run python scripts/check_docs_language.py`, `./scripts/build_docs_sites.sh`, `uv run python scripts/check_docs_nav_routes.py --check-built`, and `./scripts/build/build_api_docs.sh` when API reference is touched.
```
