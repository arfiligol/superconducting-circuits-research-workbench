# Execution & Verification

This area defines the execution and verification baseline of workspace delivery.
If a new rule is inconsistent with an existing script, the script or rule source should be corrected rather than abandoning the validation baseline.

<Aside type="note" title="What belongs here">

This layer is not defining product functions, but defining how to deliver, how to verify, and how to collaborate.
If the question is asking about build, test, CI, handoff or Codex subagent execution flow, the answer should be found here first.

</Aside>

Completion of `Implementation` does not mean completion of the entire delivery line.
integration / E2E / final summary The rules for `Testing` and `Codex Subagent Coordination` must still be aligned.

## Agent Rule
```markdown
## Execution & Verification
- Define workspace baselines for build, lint, type-check, test, and CI.
- branch roles, direct-develop policy and optional worktree use are defined by `Branch & Worktree Flow`.
- When changing the code, give priority to checks directly related to the touched area.
- The workspace delivery baseline includes five verification lines: app/frontend, app/backend, Julia Runner, desktop, and docs.
- Task scope, verification depth and subagent work lane coordination need to be aligned with the rules of this area.
```
