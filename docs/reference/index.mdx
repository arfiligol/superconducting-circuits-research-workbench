# Reference

This area is the specification sheet of research-core.

All definitions written in `Reference` should serve Pluto, Julia Core, Python Analysis Core, notebook research surfaces, research contracts, API reference and file management. The architecture, data contracts, runtime, UI, task lifecycle, and archived workflows of production applications do not belong in this area.

<Aside type="note" title="How to read Reference">

First look at the research owner layers `API Reference`, `Core`, `Julia Core`, and `Notebook Interface`; when productized application data is needed, switch to the exclusive product file lane.

</Aside>
