# Python Core

本頁記錄 active Python-owned core surfaces：`superconducting_circuits_analysis` 的 analysis package，以及 `schemdraw_circuit_library` 的 renderer-side reusable circuit visual components。

<Aside type="note" title="Owned Surfaces">

`core/python/analysis/superconducting_circuits_analysis/` owns installable Python analysis algorithms.
`core/python/circuit_libraries/schemdraw_circuit_library/` owns reusable Schemdraw visual components.
`app/backend/app_backend/domain/runtime_contracts/` owns App Backend tasking, execution, and storage runtime contracts.

</Aside>

<Aside type="caution" title="Do Not Move Ownership">

HTTP schema、ORM / repository implementation、Circuit Definition Editor preview behavior、Electron runtime concerns、Backend task runtime contracts 不得寫成 `Python Core` 的 owner 範圍。
這些屬於 App Backend、adapter 或 app layer。

</Aside>

## Owned Package Families

<Tabs>
<TabItem label="Analysis">


| Surface | Focus | Responsibility |
|---|---|---|
| `superconducting_circuits_analysis.application.analysis` | fitting, mode extraction, matrix-oriented analysis helpers | S-parameter notch/transmission/vector fitting, Y11 and SQUID-oriented fitting, and admittance mode extraction contracts |
| `superconducting_circuits_analysis.domain` | analysis value objects and schemas | trace records, fitting result semantics, and JSON-friendly analysis result shapes |

S-parameter fitting APIs accept explicit frequency arrays in Hz plus real and imaginary
trace arrays. They return plain result dictionaries with fitted parameters, metrics,
and fit/model traces so Julia Analysis Bridge and Python notebooks can consume them
without depending on notebook-local CSV files or application DTOs.

</TabItem>

<TabItem label="Schemdraw">


| Surface | Focus | Responsibility |
|---|---|---|
| `schemdraw_circuit_library.components.lumped` | lumped visual component families | ElementCompound-based resonators and related lumped reusable visuals |
| `schemdraw_circuit_library.components.transmission_lines` | transmission-line visual component families | pi sections, ladder visuals, and reducible chain drawings |
| `schemdraw_circuit_library.components.couplers` | coupling visual component families | renderer-side coupling motifs and reusable coupler components |
| `schemdraw_circuit_library.theme` | visual theme source of truth | light/dark stroke and text color selection without topology changes |
| `schemdraw_circuit_library.rendering` | render helpers | light/dark SVG/PNG variant rendering from one composition function |

Schemdraw components consume renderer-neutral data and Julia Core export hints.
They do not define circuit semantics, solver behavior, or reusable component topology.
Stable reusable components are exported through the package root; internal family modules
may hold work-in-progress visual families until their API is stable.

</TabItem>

</Tabs>

## App Backend Runtime Contracts

| Surface | Role | Why it is not Python Core |
|---|---|---|
| `app_backend.domain.runtime_contracts.tasking` | task submission kind、lane naming、dispatch routing metadata | App Backend owns product task lifecycle and Runner protocol coordination |
| `app_backend.domain.runtime_contracts.execution` | task creation、lifecycle mutation、execution operation、history / audit payload | execution records are App task-runtime contracts |
| `app_backend.domain.runtime_contracts.storage` | trace / result handles、TraceStore locator、payload lifecycle、version markers | storage publication is App Backend infrastructure and domain behavior |

Runtime contracts remain framework-agnostic inside the backend domain: no FastAPI, SQLAlchemy, file I/O, repository imports, or runner process code.

## Consumer Pairing

| Consumer | 主要依賴 |
|---|---|
| Backend task / dataset adapters | `app_backend.domain.runtime_contracts.execution`, `app_backend.domain.runtime_contracts.storage` |
| Backend runner API / publisher | `app_backend.domain.runtime_contracts.tasking`, `execution`, `storage` |
| Backend Circuit Definition Editor feature | App-local inspection and legacy netlist helpers |
| Pluto Analysis Bridge | `superconducting_circuits_analysis` |
| Docs schematic renderer | `schemdraw_circuit_library` |
