## Code Quality
- Follow Clean Code: clear naming, short functions, and single responsibility.
- UI, API, Notebook, Runner, and scripts must not each replicate business processes; sharing rules should be focused on app backend services, Julia Core, Julia Runner, or explicit contract packages.
- Prioritize fixing code rather than adding exceptions or ignoring rules.
- Consult sub-files when needed: Code Style / Type Checking / Design Patterns / Script Authoring / Data Handling / Logging / Contract Versioning / Error Handling.
